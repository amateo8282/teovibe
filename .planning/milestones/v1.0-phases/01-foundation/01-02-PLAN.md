1. 현재 config/database.yml 파일 내용을 먼저 확인하여 기존 구조 파악

2. development 섹션에 queue/cache/cable DB 추가:
   ```yaml
   development:
     primary:
       <<: *default
       database: storage/development.sqlite3
     cache:
       <<: *default
       database: storage/development_cache.sqlite3
       migrations_paths: db/cache_migrate
     queue:
       <<: *default
       database: storage/development_queue.sqlite3
       migrations_paths: db/queue_migrate
     cable:
       <<: *default
       database: storage/development_cable.sqlite3
       migrations_paths: db/cable_migrate
   ```

3. default 섹션에 pragmas가 없다면 추가하여 WAL 모드 보장:
   ```yaml
   default: &default
     adapter: sqlite3
     pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
     timeout: 5000
   ```
   - Rails 8.1 + sqlite3 2.9.0에서는 WAL이 자동 적용되지만, 새로 생성되는 Solid DB에도 적용되는지 확인 필요

4. test 섹션은 현재 상태 유지 (단일 DB)

5. production 섹션이 이미 queue/cache/cable을 포함하고 있는지 확인하고, 없으면 추가
  </action>

    - `ruby -ryaml -e "puts YAML.load_file('config/database.yml').dig('development', 'cable', 'database')"` 출력: storage/development_cable.sqlite3

<task type="auto">
  <name>Task 2: Solid Queue 활성화 및 WAL 모드 검증</name>
  <files>
    config/environments/development.rb
  </files>
  <action>
1. config/environments/development.rb 현재 내용 확인

2. Solid Queue 개발 환경 활성화 추가:
   ```ruby
   config.active_job.queue_adapter = :solid_queue
   config.solid_queue.connects_to = { database: { writing: :queue } }
   ```

3. Solid Cache/Cable은 현재 단계에서 선택적:
   - 개발에서는 memory_store(cache)와 async(cable)로 충분
   - DB 파일만 생성되고 WAL이 적용되면 Phase 1 성공 기준 충족
   - 필요 시 아래 설정 추가 (주석으로 준비):
     ```ruby
     # config.cache_store = :solid_cache_store
     # config.action_cable.cable = { "adapter" => "solid_cable" }
     ```

4. DB 생성 및 마이그레이션:
   - `bin/rails db:create:all` 실행하여 모든 DB 파일 생성
   - `bin/rails db:migrate` 실행 (queue/cache/cable 마이그레이션 포함)
   - 오류 발생 시: `bin/rails solid_queue:install:migrations`, `bin/rails solid_cache:install:migrations`, `bin/rails solid_cable:install:migrations` 후 재시도

5. WAL 모드 검증:
   - 각 DB 파일에 대해 PRAGMA journal_mode 확인:
     ```bash
     sqlite3 storage/development.sqlite3 "PRAGMA journal_mode;"
     sqlite3 storage/development_queue.sqlite3 "PRAGMA journal_mode;"
     sqlite3 storage/development_cache.sqlite3 "PRAGMA journal_mode;"
     sqlite3 storage/development_cable.sqlite3 "PRAGMA journal_mode;"
     ```
   - 모두 `wal`이어야 함
   - 만약 `delete`로 나오는 DB가 있으면: `sqlite3 {db_file} "PRAGMA journal_mode=WAL;"` 실행
  </action>
