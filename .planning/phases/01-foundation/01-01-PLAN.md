1. Gemfile 수정:
   - `gem "importmap-rails"` 제거
   - `gem "tailwindcss-rails"` 제거
   - `gem "propshaft"` 제거
   - `gem "vite_rails"` 추가
   - `bundle install` 실행

2. vite_ruby 설치 스크립트 실행:
   - `bundle exec vite install` 실행 (vite.config.ts, config/vite.json, bin/vite 등 생성)
   - 생성된 vite.config.ts를 아래 내용으로 수정:
     ```typescript
     import { defineConfig } from "vite"
     import ViteRails from "vite-plugin-rails"
     import react from "@vitejs/plugin-react"
     import tailwindcss from "@tailwindcss/vite"

     export default defineConfig({
       plugins: [
         tailwindcss(),
         react(),
         ViteRails({
           envVars: { RAILS_ENV: "development" },
           envOptions: { defineOn: "import.meta.env" },
           fullReload: {
             additionalPaths: ["config/routes.rb", "app/views/**/*"],
             delay: 300,
           },
         }),
       ],
     })
     ```

3. npm 패키지 설치 (pnpm 사용):
   - `pnpm add -D vite @vitejs/plugin-react vite-plugin-rails @tailwindcss/vite stimulus-vite-helpers`
   - `pnpm add react react-dom @hotwired/turbo-rails @hotwired/stimulus`
   - 주의: @hotwired/turbo-rails와 @hotwired/stimulus는 기존에 importmap으로 로드되던 것을 npm 패키지로 전환

4. config/vite.json 확인/수정:
   ```json
   {
     "all": {
       "watchAdditionalPaths": []
     },
     "development": {
       "autoBuild": true,
       "publicOutputDir": "vite-dev",
       "port": 3036
     },
     "test": {
       "autoBuild": true,
       "publicOutputDir": "vite-test",
       "port": 3037
     }
   }
   ```

5. Procfile.dev / bin/dev 수정:
   - Procfile.dev에서 `css: bin/rails tailwindcss:watch` 라인 제거 (있는 경우)
   - vite dev 서버 프로세스가 포함되어 있는지 확인 (bundle exec vite install이 자동 추가)
   - bin/dev가 Procfile.dev를 사용하는지 확인

6. 불필요한 파일 정리:
   - `config/importmap.rb` 삭제

     ```javascript
     import { Application } from "@hotwired/stimulus"
     import { registerControllers } from "stimulus-vite-helpers"

     const application = Application.start()
     const controllers = import.meta.glob("./**/*_controller.js", { eager: true })
     registerControllers(application, controllers)
     ```
   - 기존 `app/javascript/controllers/` 디렉토리는 삭제하지 말고 남겨둠 (안전을 위해, 검증 후 삭제 가능)


3. CSS 마이그레이션:

4. Layout 태그 교체 (`app/views/layouts/application.html.erb`):

     ```erb
     <%= vite_javascript_tag 'application' %>
     ```

   - `content_for :head` yield가 있는지 확인 (React 전용 페이지에서 사용)
