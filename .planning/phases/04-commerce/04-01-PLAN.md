  <name>Task 1: Order 모델 + 마이그레이션 생성 (SkillPack price, User payment_customer_key 포함)</name>
  <files>
    teovibe/db/migrate/XXXXXX_create_orders.rb

    teovibe/app/models/order.rb

    teovibe/app/models/user.rb
  </files>
  <action>
    1. `cd teovibe && bin/rails generate model Order user:references skill_pack:references status:integer toss_order_id:string payment_event_id:string amount:integer`으로 Order 모델 생성

    2. 생성된 마이그레이션 파일을 RESEARCH.md Pattern 1 기반으로 수정:
       - status: `default: 0, null: false` (enum: pending=0, paid=1, failed=2, refunded=3)
       - toss_order_id: `null: false`
       - amount: `null: false`
       - `add_index :orders, :toss_order_id, unique: true`
       - `add_index :orders, :payment_event_id, unique: true, where: "payment_event_id IS NOT NULL"` (SQLite에서는 partial index 지원됨)
       - `add_index :orders, :status`

    3. Order 모델(`app/models/order.rb`) 작성:
       - `belongs_to :user`, `belongs_to :skill_pack`
       - `enum :status, { pending: 0, paid: 1, failed: 2, refunded: 3 }`
       - `validates :toss_order_id, presence: true, uniqueness: true`
       - `validates :amount, presence: true, numericality: { greater_than: 0 }`
       - `before_validation :generate_toss_order_id, on: :create` (SecureRandom.urlsafe_base64(16) 사용, "order-" 접두사)

    4. SkillPack price 마이그레이션 생성: `bin/rails generate migration AddPriceToSkillPacks price:integer`
       - 마이그레이션에 `default: 0, null: false` 추가
       - SkillPack 모델에는 별도 수정 불필요 (price=0이면 무료, 0 초과이면 유료)

    5. User payment_customer_key 마이그레이션 생성: `bin/rails generate migration AddPaymentCustomerKeyToUsers payment_customer_key:string`
       - User 모델에 `before_validation :generate_payment_customer_key, on: :create` 추가
       - `generate_payment_customer_key`: `self.payment_customer_key ||= SecureRandom.uuid`

    6. `bin/rails db:migrate` 실행

    7. User 모델에 `has_many :orders, dependent: :destroy` 추가
    8. SkillPack 모델에 `has_many :orders, dependent: :destroy` 추가
  </action>
  <verify>

  </verify>

<task type="auto">
  <name>Task 2: 체크아웃 라우트 + CheckoutsController 스켈레톤 생성</name>
  <files>
    teovibe/config/routes.rb
    teovibe/app/controllers/checkouts_controller.rb
  </files>
  <action>
    1. `config/routes.rb`의 skill_packs 리소스 블록에 체크아웃 라우트 추가 (RESEARCH.md Pattern 4 참고):
       ```ruby
       resources :skill_packs, only: [:index, :show] do
         member do
           get :download
           get "checkout", to: "checkouts#show", as: :checkout
           get "checkout/success", to: "checkouts#success", as: :checkout_success
           get "checkout/fail", to: "checkouts#fail", as: :checkout_fail
         end
       end
       ```

    2. `app/controllers/checkouts_controller.rb` 생성:
       - `before_action :require_authentication` (Rails 8.1 인증 기본 패턴)
       - `before_action :set_skill_pack`
       - `show`: pending Order를 find_or_create_by(user: Current.user, skill_pack: @skill_pack, status: :pending) + amount 설정. User의 payment_customer_key가 없으면 생성 후 저장. `@order`, `@client_key`, `@customer_key` 인스턴스 변수 설정. client_key는 `Rails.application.credentials.dig(:toss_payments, :client_key)` 사용.
       - `success`: paymentKey, orderId, amount 파라미터 수신. 금액 검증 후 Plan 03에서 confirm 로직 추가 예정이므로 현재는 "결제 확인 처리 중" 안내만 표시.
       - `fail`: code, message 파라미터를 flash에 담아 에러 표시.
       - `set_skill_pack`: `@skill_pack = SkillPack.published.find(params[:skill_pack_id] || params[:id])`

    3. price가 0인 스킬팩은 체크아웃 불필요하므로 show 액션 시작에 `redirect_to skill_pack_path(@skill_pack), alert: "무료 스킬팩입니다."` 가드 추가 (`return if @skill_pack.price.zero?`)
  </action>
