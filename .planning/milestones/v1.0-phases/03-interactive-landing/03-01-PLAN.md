       - `api/` 디렉토리와 `api/v1/` 디렉토리가 없으면 생성

  <action>
    1. React 진입점 생성 — `teovibe/app/frontend/entrypoints/landing.jsx`:
       - `react-demo.jsx`와 동일한 turbo:load + turbo:before-cache 패턴 재사용
       - module 스코프에 `let root = null` 저장
       - turbo:load: `document.getElementById("landing-root")`에 마운트 (`if (el && !root)` 가드)
       - turbo:before-cache: unmount + `root = null`
       - import `LandingPage` from `../components/landing/LandingPage`

    2. LandingPage 컨테이너 생성 — `teovibe/app/frontend/components/landing/LandingPage.tsx`:
       - TypeScript 인터페이스 정의:
         ```typescript
         interface SectionCard {
           title: string; description: string; icon: string;
           link_url: string; link_text: string; position: number;
         }
         interface LandingSection {
           id: number; section_type: string; title: string; subtitle: string;
           background_color: string | null; text_color: string | null;
           section_cards: SectionCard[];
         }
         ```
       - `useState<LandingSection[]>([])` + `useState(true)` for loading
       - `useEffect`에서 `fetch("/api/v1/landing_sections")` -> `setSections(data)` -> `setLoading(false)`
       - 로딩 중: `<div className="min-h-[744px] bg-tv-cream" />` (CLS 방지)
       - 섹션 없을 때: DefaultHero 컴포넌트 (기존 home.html.erb의 fallback 히어로를 React로 포팅. `link_to` 대신 `<a href="/registrations/new">` 등 하드코딩 경로 사용)
       - SECTION_COMPONENTS 맵: `{ hero: HeroSection }` (이 플랜에서는 hero만. 나머지는 Plan 02에서 추가)
       - 알 수 없는 section_type은 `null` 반환 (렌더링 스킵)

    3. HeroSection 생성 — `teovibe/app/frontend/components/landing/HeroSection.tsx`:
       - `import { motion } from "motion/react"` (framer-motion 아님!)
       - 기존 `_hero.html.erb` 디자인을 그대로 포팅:
         - `min-h-[744px]`, `bg-tv-cream`, 그리드 SVG 배경 (style prop으로 backgroundImage)
         - 제목: `text-display md:text-hero font-black tracking-tight`
         - 부제목: `text-lg md:text-xl text-tv-gray`
         - CTA 버튼: `<a href="/registrations/new">시작하기</a>`, `<a href="/about">더 알아보기</a>`
       - motion 애니메이션 적용:
         - containerVariants: `hidden: { opacity: 0 }`, `show: { opacity: 1, transition: { staggerChildren: 0.15, delayChildren: 0.1 } }`
         - itemVariants: `hidden: { opacity: 0, y: 24 }`, `show: { opacity: 1, y: 0, transition: { duration: 0.6, ease: [0.22, 1, 0.36, 1] } }`
         - h1, p, CTA div를 각각 `<motion.h1 variants={itemVariants}>` 등으로 감싸기
       - props: `{ section: LandingSection }` 타입 (LandingPage에서 import하는 타입)

    4. home.html.erb 교체:
       - 기존 ERB 섹션 렌더링 코드를 React 마운트 포인트로 교체
       - 기존 코드를 모두 제거하고 다음으로 대체:
         ```erb
         <div id="landing-root"></div>

         <%% content_for :head do %>
           <%%= vite_javascript_tag 'landing' %>
         <%% end %>
         ```
       - 주의: 기존 ERB 파셜(`_hero.html.erb` 등)은 삭제하지 말 것. 참조용으로 유지.

    5. `components/landing/` 디렉토리가 없으면 생성
