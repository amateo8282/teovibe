   - `enum :record_type, { post: 0, skill_pack: 1 }`
   - `has_many :posts, foreign_key: :category_id` (record_type=post)
   - `has_many :skill_packs, foreign_key: :category_id` (record_type=skill_pack)
   - `validates :name, presence: true`
   - `validates :slug, presence: true, uniqueness: { scope: :record_type }`
   - scopes: `ordered` (position asc), `for_posts`, `for_skill_packs`, `visible_in_nav`
   - `before_destroy :check_associated_records` — 게시글/스킬팩이 있으면 삭제 거부 ("게시글 N개가 있어 삭제할 수 없습니다")

   - unique index: [:slug, :record_type]
   - Post 카테고리 6개 시드 (blog, tutorial, free-board, qna, portfolio, notice). notice는 admin_only: true
   - SkillPack 카테고리 4개 시드 (template, component, guide, toolkit)
   - posts에 category_id 컬럼 추가
   - **slug 기반 서브쿼리로 enum→FK 매핑** (STATE.md 결정: auto-increment ID 가정 금지)
     - `UPDATE posts SET category_id = (SELECT id FROM categories WHERE slug = 'blog' AND record_type = 0) WHERE category = 0` 패턴
   - posts.category enum 컬럼 제거 (Rails 8.x + SQLite 3.35+ 지원 확인)
   - skill_packs 동일 처리 (category_id 추가, enum→FK 매핑, category enum 컬럼 제거)
   - `down` 메서드: `raise ActiveRecord::IrreversibleMigration`
   - **마이그레이션 후 데이터 검증**: 매핑 전후 레코드 수 일치 확인 로직 포함
